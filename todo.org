* trader/data cleanup checklist
** Goal
The immediate goal is to incrementally clean up the current candle
storage, inventory, planner, and build-loop issues.

For each fix:
- make one focused change
- compile
- run
- verify behavior
- then move to the next item

Only after cleanup is stable should we move on to improving build-task
accuracy, planner correctness, and pipeline efficiency.

** Cleanup strategy
*** Rules
- [ ] Make one change at a time
- [ ] Compile after every change
- [ ] Run after every change
- [ ] Do not mix cleanup with feature improvements
- [ ] Prefer deleting duplicate code over keeping parallel paths alive
- [ ] Keep notes on any behavior changes observed after each cleanup item

*** Current guiding principle
Clean up and stabilize first.
Then improve planning accuracy, build correctness, and performance.
