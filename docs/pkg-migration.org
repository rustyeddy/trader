* Goal

Break the current large root =trader= package into well-defined
packages with clear dependency direction, while keeping behavior
stable and avoiding large disruptive rewrites.

The migration should move from low-level, low-dependency concepts
outward toward orchestration code. In practice, that means shrinking
the root package from the leaves inward.

* Wave 8 (in progress): de-shim the root engine

The backtest/engine orchestration still lives in the root =trader= package, but
it no longer rides on the =execution=/=strategy= re-export shims:

- The root engine (=trader.go=, =backtest*.go=, =app_config.go=, =live_strategy_registry.go=,
  and their tests) now references =execution.X= and =strategy.X= directly. Done
  with an AST-safe =gofmt -r= qualification pass (string-literal-safe; the
  field-name / keyed-literal-key damage the doc's Lesson 4 warns about was
  compiler-caught and reverted), plus =goimports=. Two local vars named
  =strategy= that shadowed the new package import were renamed to =strat=.
- =execution_aliases.go= and =strategy_aliases.go= are DELETED. The last external
  caller still on them (=cmd/bot=, via an aliased =traderpkg= import the earlier
  =trader.=-grep missed) was repointed to =strategy= directly.

External-caller shim migration (DONE): all external callers of the
=market=/=indicator=/=journal= re-export shims have been migrated to import those
packages directly. Worked in three buckets by coupling:

- Bucket A (34 files): referenced ONLY =market= via the shim. Pure import swap +
  =trader.= -> =market.=; the root import is dropped entirely.
- Bucket C (20 files): =market= primitives alongside genuine root engine types
  (=Backtest=, =RootConfig=, =Live*=). Qualified the market selectors with an
  AST-safe =gofmt -r= pass; the =trader= import is RETAINED for the engine types.
- Bucket B (7 files): =market= plus the =indicator= and/or =journal= shims.
  Qualified all three layers, which lets these files drop the =trader= import.

With Bucket B landed, =indicator_aliases.go= and =journal_aliases.go= became dead
(no external OR root caller) and were DELETED. =roundDivPositive= had already
moved out with ChandelierExit, so the indicator shim had no root user left.

Log shim removed (DONE): =log_aliases.go= is DELETED. Assessment found the root
engine's only genuine log usage was the module logger =L= in =trader.go= (14
call sites; everything else flagged by the raw grep was config struct fields
=Data/Replay= or org-table string literals). Qualified =L= -> =log.L= with
=gofmt -r= + =goimports= (which correctly resolved the trader =log= package over
stdlib =log=, since no file imported stdlib =log=). Five external callers
migrated to =log.X=: =cmd/main.go=, =cmd/serve/cmd_serve.go=,
=cmd/mcp/cmd_mcp.go=, =cmd/backtest/cmd_backtest.go=, =service/sweep_test.go=
(the first two used the =traderpkg=-aliased and bare =trader.= forms
respectively — Lesson 6 again).

Market shim removed (DONE): =market_aliases.go= is DELETED. This was the last
remaining transition shim — NO =*_aliases.go= files exist anywhere now. The root
engine used its 78 market symbols BARE (exported types =Price/Candle/...= plus
root-internal unexported helpers =mulDivCeil64/bitSet/...=). Qualified all of
them across every top-level root file (production + tests) with an AST-safe
=gofmt -r= pass driven by the shim's own LHS->RHS map (17 of 78 differed: the
two lowercase candle aliases and the unexported helper set map to Exported
=market.= names), then =goimports=. Per Lesson 4, the only collateral damage was
keyed composite-literal KEYS (=Instrument:= -> =market.Instrument:=); =gofmt -r=
leaves struct field declarations, selectors, type decls, and switch =case= values
correct. The damaged keys were reverted programmatically off the compiler's
"invalid field name market.X in struct literal" errors (build for production
files, =go vet= for test files), then =goimports= dropped now-unused =market=

With this, the root engine no longer rides on ANY re-export shim.

Still remaining for wave 8: the engine/backtest split (see next section). All
leaf-package qualification (execution/strategy/market/indicator/journal/log) is
DONE, so this is no longer blocked on import-name cleanup.

Remaining in root: engine + backtest run orchestration, live orchestration,
config, and a few orphans.

* Wave 8 follow-up (PLANNED): split engine and backtest into sibling packages

DECIDED layout — Option A: =engine= and =backtest= become sibling directories.
Root =trader= stops being the engine; the two new packages sit side by side with

#+begin_src text
trader/
  engine/      package engine    — Trader + broker/event loop (mechanism only)
  backtest/    package backtest  — Backtest* run orchestration; imports engine
#+end_src

** Four-way classification of the 16 remaining root files

The current root package is NOT one layer; it is four concerns tangled together.

1. ENGINE (mechanism, low-level) — the part of =trader.go= that is the =Trader=
   type and its broker/loop plumbing:
   - =Trader= struct (=DataManager= + =execution.Broker= + =marketdata.Store=)
   - =startBrokerEventHandler=, =brokerEventError=, =waitForBrokerIdle=,
     =processEvent=, =snapshotLots=, =eventPositionID=
   Pure mechanism: drive a broker, drain events, track lots. No notion of a run.

2. BACKTEST (higher-level run orchestration) -> new =backtest= package:
   - all nine =backtest_*.go= files PLUS =backtest_config.go=
     (=Config=/=RunConfig=/=RunDefaults=/=DataConfig=/=LoadConfig=/=hashBacktestConfig=)
   - PLUS the two loop methods currently MIS-HOMED on =Trader=:
     =Backtest()= and =backTestWithIterator()=. These reference =*Backtest=, so
     they are the inversion point (below).

3. LIVE (higher-level, separate concern) — =live_strategy.go= (=LiveStrategy=,
   =LivePrice=, =LiveTrade=, =LivePlan=, =LiveOpenRequest=) and
   =live_strategy_registry.go=. Depends on =strategy=, not on backtest. Its own
   eventual home (=live= package, or fold into =strategy=/=service=) is a SEPARATE
   decision from the engine/backtest split — out of scope here.

4. CONFIG / ORPHAN — =app_config.go= (=RootConfig=, =appConfig=, ...),
   =config_global.go= (=GlobalConfig=, ...), and =version.go=. App/global config,
   not engine. Stay put for now.

** The gating prerequisite: invert =Trader.Backtest=

The one thing that makes =trader.go= LOOK fused is that the run loop hangs off the
engine type and references the run value:

#+begin_src go
func (t *Trader) Backtest(ctx, run *Backtest) error
func (t *Trader) backTestWithIterator(ctx, run *Backtest, itr market.CandleIterator) error
#+end_src

If =backtest= becomes the higher package, this edge points the WRONG way
(engine -> backtest -> engine = cycle). So the split's real work is inverting it:
move =Backtest()= / =backTestWithIterator()= out of =Trader= and into the
=backtest= package as functions/methods that take a =*engine.Trader= (or a small
engine interface) and call its lower-level primitives. =Trader= is then left with
only the broker/store/loop mechanism, and the dependency is cleanly one-way.

Once inverted, the physical move is just =git mv= + package rename for each side.

** External-caller fallout (same pattern as the rest of wave 8)

- =backtest_config.go= defines =Config=/=RunConfig=, consumed by =service=
  (=CompileBacktests=, the executor) and =cmd/backtest=. When it moves, those
  callers re-point to =github.com/rustyeddy/trader/backtest= — one more
  external-caller migration like buckets A/B/C.
- Anything constructing or driving =Trader= (notably the service backtest
  executor) re-points to =github.com/rustyeddy/trader/engine=.
- A transition shim (=engine_aliases.go= / =backtest_aliases.go=) MAY be used to
  stage callers, but prefer a direct cutover now that the qualification toolkit
  (=gofmt -r= + =goimports=, Lessons 4 and 6) is well-worn.

* Lessons Learned (waves 1-3)

These corrections supersede the original plan where they conflict.

1. *Treat the candle family as part of the primitives wave, not split.*
   The original plan put =candle_types.go= in wave 1 but =candle_agg/gap/iter/set=
   in wave 4. That split is wrong: they are one cohesive unit and were moved
   together into =market=. More importantly, indicators depend on =Candle=, and
   root depends on indicators, so =Candle= MUST be extracted before indicators or
   the move is an import cycle. Candle is a hard prerequisite, not a wave-4
   afterthought.

2. *The real work is unexported helpers crossing the new boundary, not the
   leaf types.* Type aliases handle the public surface trivially. The effort
   went into the additive export layer each new package needs for the
   unexported internals the old package still calls: =mulDiv*=, =bitSet/bitIsSet=,
   =candleSet/candleSetIterator=, =roundDivPositive=, the =Source= constants,
   =formatScaledPrice=, etc. Budget for a =migration_exports.go= in every package.

3. *=journal_live= did not need to be deferred.* It depends only on
   =brokers/oanda=, which does not import root, so there is no cycle. It moved in
   the same PR as the rest of journal. Apply the same test before deferring any
   "live/OANDA" file: check whether its only non-stdlib dependency imports root.

4. *The mechanical =market.=-qualification pass inside a moved package has three
   traps.* A naive =s/\bPrice\b/market.Price/= will wrongly rewrite: (a) struct
   FIELD names that match a type name (=Units Units=), (b) keyed composite-literal
   keys (=Units:=), and (c) it will MISS helper functions that also need
   qualifying (=PriceFromFloat=, =FromTime=, =ShortDisplayID=, =PriceScale=,
   =FormatScaledPrice=). Qualify types AND helpers; then revert field-name and
   literal-key damage; then build.

5. *One self-inflicted wart to clean up in wave 4.* Moving =candleSet= into
   =market= separated the candle storage TYPE from the storage LOGIC (=store.go=,
   =data_manager.go=, ~12 root files) that still drives it through shims. Wave 4
   is partly "reunite candleSet with its users." Net: store/data has almost no
   UPWARD coupling (one stray =Trader= reference total), so the difficulty is
   volume and this reunion, not entanglement with the engine.

6. *Import ALIASES defeat the =trader.=-prefix grep that scopes a caller
   migration.* Several =cmd/*= files import root as =traderpkg
   "github.com/rustyeddy/trader"=, so they call =traderpkg.TradeRecord=, not
   =trader.TradeRecord=. A bucket analysis built on =grep "trader\."= silently
   omits them, and the gap only surfaces as a compile error when the shim is
   deleted (=cmd/bot/cmd_bot.go= hid a journal-shim caller this way). Before
   scoping AND before declaring a shim removable, grep for the package's symbols
   under EVERY local import name, not just the default =trader.=. Cheapest robust
   form: grep the bare symbol set across the tree, or enumerate the alias names
   first (=grep -rE '[a-z]\w* "github.com/rustyeddy/trader"'=) and search each.

* Recommended Dependency Direction

Target shape (extracted layers in brackets):

#+begin_src text
cmd, api/rest, api/mcp
        |
     service
        |
 backtest / live orchestration        <- engine + strategy/regime
        |

        |

        |

Lower-level packages must not import higher-level packages. Domain types must

** Resolved: Trade / Position / Lot placement

The original plan was ambiguous (listed under both wave 1 "Later" and wave 5).
Resolved by inspection: =trade.go= couples to =closeCause= and
=trade_request.go= couples to =OpenRequest=, both accounting/execution concepts.
=Trade=, =Position=, and =Lot= belong in the *execution* package, NOT =market=.

** Reorder: execution is a clean leaf on =market=

=account.go=, =broker.go=, =trade*.go=, =trade_request.go= form a cohesive
cluster that references each other plus =Candle= (now in =market=). It has NO
upward dependency on Backtest, Strategy, Journal, or DataManager. So execution
can be extracted relatively early, right after marketdata (or even alongside).
The "delicacy" the original plan warned about is about INVARIANTS (equity,
margin, P/L), not dependency direction. Test coverage must be the gate.

** Key rework: break the strategy -> Backtest hard dependency

=Strategy.Update(ctx, *CandleTime, *Backtest)= passes the concrete =*Backtest=,
which is why the strategy layer is welded to the engine. But the actual usage is
THIN and specific: strategies only read =run.Request.Instrument= (which
instrument) and =run.State.Lots= (current open positions). Most strategies
ignore the argument entirely.

Proposal: introduce a narrow interface, e.g.

#+begin_src go
type StrategyContext interface {
    Instrument() string
    OpenLots() LotView   // read-only view of current positions
}
#+end_src

=Backtest= implements it; =Strategy.Update= takes =StrategyContext= instead of
=*Backtest=. This decouples the entire strategy/regime layer from the engine and
lets =strategy= be extracted WITHOUT dragging =backtest= along. This is the
single highest-leverage change for untangling wave 6, and it is small because
the real surface is two methods. (Nothing is sacred — this is the rework the
thin dependency invites.)

* Wave 4: the marketdata arrangement (next)

Detailed design for the store/data extraction, derived from re-assessing the
actual dependency graph.

** Findings

- store/data (~4,500 lines core + ~870 lines stats) has ZERO upward
  dependencies — a clean leaf on =market=.
- =candleSet= + =candle_agg/gap/iter= (~500 lines, currently in =market=) are
  used ONLY by the store/loader, the candle files themselves, and
  =testdata_generator=. The engine/strategy code never touches =candleSet= — it
  only uses =Candle=, =CandleTime=, and =CandleIterator=.
- =data/= is already a dependency-free =Provider= interface + registry.
  =data/dukascopy= registers there and depends UP on root's =Store=/=Key=. Root
  fetches providers via =data.Get()= and never imports dukascopy (blank-imported
  in =cmd=). Preserve this inversion.
- =Store= reads/writes =candleSet= (store.go uses it 5x) and =DataManager= uses
  =Store= 14x — one cohesive subsystem; do not split =store= out (it would cycle
  with =candleSet=).
- =data_stats_*= depend only on =Candle=/=CandleTime=/=CandleIterator=, but they
  are folded INTO =marketdata= (decision: one package, clearly named).

** Two structural decisions

1. *candleSet reunion*: move =candleSet= + =candle_agg/gap/iter= OUT of =market=
   and INTO =marketdata=. =candleSet= is storage machinery (monthly files,
   validity bitmaps, gap reports), not a value primitive. =market= keeps only
   the pure =Candle= value. Deletes the =market= candleSet shims.

2. *CandleIterator -> market*: lift the =CandleIterator= interface (yields
   =CandleTime=) into =market=. The engine CONSUMES iterators; =marketdata=
   PRODUCES them. With the interface in =market=, the engine depends on
   =market.CandleIterator= and not on =marketdata= just for the contract. Same
   inversion pattern as =StrategyContext=. Concrete impls stay in =marketdata=.

** Lessons from the big move

- Qualifying a large package against =market.= has more traps than the small
  ones: besides field names / literal keys / method names, watch (a) =market.X=
  landing inside STRING LITERALS (corrupted a CSV header and test fixtures —

* Practical Migration Rules

- Keep behavior unchanged during package moves.
- Use type aliases in the old root package temporarily to keep churn manageable;
  add a =migration_exports.go= in each new package for unexported helpers the old
  package still calls. Delete shim entries as callers migrate to the new import.
- After the =market.=-qualification pass, audit for the three traps in Lesson 4.
- Move tests with their packages; verify moved =_test.go= files do not import the
  root package (that is an import cycle once root imports the new package).
- Prefer one package boundary per PR or commit.
- Run =gofmt=, =go vet ./...=, =make test=, and =make sweep= after each move.
- Preserve the no-floats internal calculation rule throughout the migration.
- Avoid putting APIs, CLI handlers, or OANDA-specific wire concerns into domain
  packages.

* Notes

Do not start by splitting =service/=. It already acts as an application boundary
between the transport layers and the core engine.

The remaining hard problem is the engine/strategy knot. The =StrategyContext=
rework (step 7) is what makes it tractable: once strategies no longer take
=*Backtest=, the strategy/regime layer and the engine can be separated cleanly
instead of being moved together into multiple directories.
