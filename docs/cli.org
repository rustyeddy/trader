* trader — Backtesting, replay, and live trading tooling

Top-level flags apply to all commands.

- =--data-dir=    Root directory for candle data (default =/srv/trading/data/candles=)
- =--log-level=   Log level: debug|info|warn|error (default =debug=)
- =--log-format=  Log format: text|json (default =text=)
- =--log-file=    Path to log file written in addition to stdout
- =--no-color=    Disable coloured output

** backtest — Run and inspect historical backtest results

Run YAML-configured backtests and inspect saved results.  Reports are
stored as JSON + org-mode files whose filenames are derived from a hash
of the run parameters; re-running the same config overwrites the same
file rather than creating a timestamped copy.

*** backtest run — Run backtest configs and write JSON + org reports

Run one or more YAML backtest configs.  Changing any parameter produces
a distinct report file alongside previous ones.

Config and result directories default to
=$TRADER_BACKTEST_DIR/{configs,reports}= (falling back to
=/srv/trading/backtests/{configs,reports}=).

#+BEGIN_EXAMPLE
trader backtest run
trader backtest run myconfig.yml
trader backtest run --config /path/to/configs --out /path/to/reports
#+END_EXAMPLE

Flags:
- =--config=  Backtest config file, directory, or glob
- =--out=     Output directory for reports

*** backtest list — List saved backtest results

List saved backtest JSON reports from the reports directory.  Filter by
instrument or strategy name.

Flags:
- =--dir=          Reports directory
- =--instrument=   Filter by instrument (case-insensitive substring)
- =--strategy=     Filter by strategy name (case-insensitive substring)

*** backtest get — Show full details for a saved backtest result

Print the full summary for a named backtest report.  The name argument is
the report filename without the =.json= extension.

Flags:
- =--dir=  Reports directory

*** backtest candles — Print OHLC candles for a saved backtest result as CSV

Print the OHLC candle data for the instrument/timeframe/range described in
a named backtest report.  Output is CSV with columns:
=time,open,high,low,close=.

Flags:
- =--dir=  Reports directory

*** backtest org — Print the org-mode report for a saved backtest result

Print the org-mode formatted report for a named backtest result to stdout.

Flags:
- =--dir=  Reports directory

*** backtest configs — List available backtest config files

List YAML/JSON backtest config files in the configs directory.

Flags:
- =--dir=  Configs directory

*** backtest regress — Compare backtest results against committed baselines

Compare current backtest results against committed baseline JSON files
and exit 1 on any regression.  Use =--update= to write the current
results as new baselines instead of comparing.

Flags:
- =--baselines=  Directory containing committed baseline JSON reports (default =testdata/backtests/reports=)
- =--update=     Write current results as new baselines

** bot — Manage live strategy bots

Start, stop, inspect, and report on live strategy bots.  By default all
subcommands are thin HTTP clients that talk to a running =trader serve=
process via the REST API (=--server= controls the URL).

For ad-hoc or development use, =bot start --local= runs a bot directly in
the CLI process without requiring a server.

*** bot start — Start one or more live strategy bots

Start one or more live strategy bots.  Supply either individual flags for
a single bot or =--config= for a portfolio YAML (one bot per instrument).

OANDA positions are NOT closed on stop — they remain on the broker.

#+BEGIN_EXAMPLE
# server mode (bot runs inside trader serve)
trader bot start --instrument EUR_USD --strategy pulse
trader bot start --config testdata/configs/smoke-test.yml

# local mode (runs directly in this process, no server required)
trader bot start --local --instrument EUR_USD --strategy pulse --token $TOKEN
#+END_EXAMPLE

Flags:
- =--instrument=     Instrument, e.g. EUR_USD
- =--strategy=       Strategy name, e.g. donchian-v4, bb-fade, pulse
- =--config=         Portfolio YAML config (starts one bot per instrument)
- =--risk-pct=       Percent of account equity to risk per trade (default 1)
- =--max-units=      Maximum position size in units (0 = no limit)
- =--tick-interval=  Poll interval, e.g. 30s, 5m (default 60s)
- =--local=          Run bot directly in this process (no =trader serve= required)
- =--token=          OANDA API token (local mode)
- =--account-id=     OANDA account ID (local mode)
- =--env=            OANDA environment: practice|live (local mode, default practice)

*** bot stop — Stop a running bot

Stop a running bot by ID.  The OANDA position is left open on the broker.

*** bot list — List all bots (running and stopped)

List all bots known to the server, both currently running and stopped,
with their instrument, strategy, tick count, and status.

*** bot get — Show status of a single bot

Show the current status of one bot by ID, including runtime stats.

*** bot report — Show a detailed run report for one or all bots

Show a detailed report for a bot run including metadata, runtime stats,
and P&L.  P&L data is read from the journal file; only trades tagged with
a bot ID (those opened after bot tracking was introduced) appear in the
P&L section.

Flags:
- =--all=      Report all bots (running and stopped)
- =--journal=  Path to JSONL trades journal for P&L data (default =./live-trades.jsonl=)

** data — Download, build, inspect, and validate candle data

Manage the local candle store used by backtests and live warmup.  Candles
are stored at =--data-dir= (default =/srv/trading/data/candles=).

*** data update — Catch-up download: fill OANDA candle data from last stored date

Download OANDA candles for every instrument+timeframe, starting from the
day after the last non-zero candle on disk and ending at yesterday (last
complete UTC day).  All 24 tracked instruments and four timeframes (M1,
H1, H4, D) are updated by default.

#+BEGIN_EXAMPLE
trader data update
trader data update --instruments EUR_USD,GBP_USD
trader data update --timeframes H1,D
trader data update --dry-run
#+END_EXAMPLE

Flags:
- =--instruments=  Comma-separated instruments to update (default: all 24)
- =--timeframes=   Comma-separated timeframes (default: M1,H1,H4,D)
- =--dry-run=      Print what would be downloaded without fetching
- =--from=         Seed start date (YYYY-MM-DD) when no prior data exists
- =--token=        OANDA API token
- =--env=          OANDA environment: practice|live (default practice)

*** data oanda — Download candles from OANDA into the canonical candle store

Download a specific instrument/timeframe/date-range directly from OANDA
and write it into the canonical candle store.

Flags:
- =--instrument=  OANDA-format instrument, e.g. USD_JPY (required)
- =--timeframe=   M1, H1, or D (required)
- =--from=        Start date YYYY-MM-DD (required)
- =--to=          End date YYYY-MM-DD (required)
- =--token=       OANDA API token
- =--raw-dir=     Root for raw bid+ask candle preservation

*** data candles — Print local candles in canonical CSV format

Print stored candles for an instrument/timeframe/date-range to stdout as
CSV (=time,open,high,low,close=).

Flags:
- =--instrument=  FX pair, e.g. EURUSD
- =--timeframe=   M1, H1, or D1 (default H1)
- =--from=        Start date inclusive (YYYY-MM-DD)
- =--to=          End date inclusive (YYYY-MM-DD)

*** data stats — Print swing, spread, trend, and session statistics

Walk a candle dataset and report four distribution groups:

- Swing Range: high-low range in pips
- Avg Spread: spread distribution in pips
- Trend: body/range ratio (trending vs consolidating bars)
- Session: average range and candle count by UTC hour

Flags:
- =--instrument=  FX pair (e.g. EURUSD)
- =--timeframe=   M1, H1, or D1 (default H1)
- =--from=        Start date inclusive (YYYY-MM-DD)
- =--to=          End date inclusive (YYYY-MM-DD)
- =--units=       Show USD value per N units alongside pips

*** data sync — Download ticks and build candles (combined)

Download missing Dukascopy tick files and then build candles from them
in one step.

Flags:
- =--instruments=  Comma-separated instruments
- =--from=         Start month inclusive (YYYY-MM)
- =--to=           End month inclusive (YYYY-MM)

*** data download-ticks — Download missing tick files

Download missing Dukascopy tick files for the given instruments and
month range without building candles.

Flags:
- =--instruments=  Comma-separated instruments
- =--from=         Start month inclusive (YYYY-MM)
- =--to=           End month inclusive (YYYY-MM)

*** data build-candles — Build candles from existing tick data

Build candles from already-downloaded Dukascopy tick files for the
given instruments and month range.

Flags:
- =--instruments=  Comma-separated instruments
- =--from=         Start month inclusive (YYYY-MM)
- =--to=           End month inclusive (YYYY-MM)

*** data validate-candles — Scan stored candle months for missing slots and source mismatches

Scan stored candle months for missing expected time slots and optionally
compare canonical candles against raw source data to detect corruption.

Flags:
- =--instruments=  Comma-separated instruments
- =--timeframe=    Candle timeframe to validate (default H1)
- =--from=         Start month inclusive (YYYY-MM)
- =--to=           End month inclusive (YYYY-MM)
- =--check-raw=    Also compare canonical candles with raw source (default true)
- =--report=       Optional JSON report output path

*** data pip-value — Show USD value of pips for each major pair

Print the USD value of 1, 10, 100, and 1000 pips for every major FX pair
at the given position size (default: 100,000 units = 1 standard lot).
Rates are resolved from =--rates= flag, live OANDA prices, or built-in
approximate defaults.

Flags:
- =--units=   Position size in units (default 100000)
- =--rates=   Explicit rates for USD-base pairs, e.g. =USDJPY=152.50,USDCHF=0.88=
- =--token=   OANDA API token for live rate lookup
- =--env=     OANDA environment (default practice)

*** data position — Convert between position size, notional value, and pip P&L

Calculate USD notional value, margin required, and pip P&L for a position.
Without =--units= or =--notional= prints a reference table for
micro/mini/standard lots.

Flags:
- =--instrument=  FX pair (required)
- =--units=       Show notional and margin for N units
- =--notional=    Show how many units are needed for $N notional
- =--pips=        Add a column showing the USD value of N pips
- =--price=       Current mid price (fetched live from OANDA if omitted)
- =--token=       OANDA API token for live price lookup

** docs — Generate CLI reference documentation

Generate Markdown (or man/RST) reference documentation for all trader
commands using Cobra's built-in doc generator.  By default all commands
are written to a single combined file.

#+BEGIN_EXAMPLE
trader docs                          # writes docs/trader-cli.md
trader docs --file /tmp/cli.md
trader docs --single=false --out ./docs/cli/
trader docs --format man --out ./man/
#+END_EXAMPLE

Flags:
- =--single=   Write all commands to one combined file (default true)
- =--file=     Output file path when =--single= is set (default =docs/trader-cli.md=)
- =--out=      Output directory when =--single=false= (default =./docs=)
- =--format=   Output format: markdown, man, rst (default markdown)

** health — Check the health and version of a running trader serve

Probe a running =trader serve= instance over HTTP.

Flags:
- =--server=  Base URL of the =trader serve= instance (default =http://localhost:8080=)

*** health check — Ping the health endpoint and print status

Ping the =/api/v1/health= endpoint and print the response status.

*** health version — Print the version reported by a running trader serve

Fetch and print the version string from a running =trader serve= instance.

** live — Live trading subsystem

Live-data utilities.  Strategy bots are managed via =trader bot=.

*** live journal — Subscribe to OANDA transactions and journal closed trades

Subscribe to the OANDA transaction push stream and write closed trade
records and equity snapshots to JSONL journal files.  =trader serve=
runs this automatically with reconnect logic; use this command for
standalone journalling without the full daemon.

Flags:
- =--token=          OANDA API token
- =--account-id=     OANDA account ID (auto-discovered if omitted)
- =--env=            OANDA environment: practice|live (default practice)
- =--journal=        Journal backend: csv|json (default json)
- =--trades-file=    Path for trade records (default =live-trades.jsonl=)
- =--equity-file=    Path for equity records (default =live-equity.jsonl=)
- =--backfill-from=  If >0, poll =GetTransactions= from this ID before streaming

** mcp — MCP server exposing trader as typed Claude tools (stdio transport)

Start the trader MCP server using stdio transport.  Designed to be
spawned as a subprocess by Claude Code or Claude Desktop — not a daemon.
This is distinct from the HTTP MCP endpoint exposed by =trader serve=
(=POST /mcp=), which is for network-connected Claude clients.

Add to =~/.claude/mcp_servers.json= or your Claude Desktop config:
#+BEGIN_EXAMPLE
{
  "trader": {
    "command": "trader",
    "args": ["mcp"]
  }
}
#+END_EXAMPLE

Without =--token= only backtest tools are available.  With =--token= live
account and trade tools are enabled.  Write tools require =--enable-write=.

Read-only tools:
- =get_account_summary=  Current balance, NAV, margin, P/L
- =list_open_trades=     All open OANDA positions
- =get_transactions=     Account transaction history
- =run_backtest=         Run YAML config(s) and return summaries
- =list_bots=            List all live strategy bots
- =get_bot=              Get status of a single bot by ID

Additional write tools (=--enable-write=):
- =place_order=          Size and submit a market order
- =close_trade=          Close a position fully or partially
- =update_stop=          Move stop-loss or take-profit
- =start_bot=            Start a new live strategy bot
- =stop_bot=             Stop a running bot by ID

Resources:
- =backtest://results=   List or read backtest org reports
- =config://configs=     List or read YAML backtest configs

Flags:
- =--token=         OANDA API token (enables live endpoints)
- =--account-id=    OANDA account ID
- =--env=           OANDA environment: practice|live (default practice)
- =--enable-write=  Enable write tools
- =--reports-dir=   Backtest reports directory

** order — Live order management (OANDA)

Direct OANDA account and order management commands.  All subcommands
require =--token= (or =OANDA_TOKEN= in the environment).

*** order account — Print account balance, NAV, margin, and unrealized P/L

Fetch and display the current OANDA account summary.

*** order new — Size and submit a market order with confirmation

Size a market order based on account equity and risk percentage, then
submit it to OANDA with a confirmation prompt.

Flags:
- =--instrument=  Instrument in OANDA format, e.g. USD_JPY (required)
- =--side=        Trade direction: long or short (required)
- =--stop-pips=   Stop distance in pips (required)
- =--risk-pct=    Percent of account equity to risk per trade (default 1)

*** order list — List open trades from OANDA

Fetch and display all currently open trades on the OANDA account.

*** order close — Close an open trade (full or partial)

Close an open trade fully or partially by trade ID.

Flags:
- =--trade-id=  Trade ID to close (required)
- =--units=     Units to close (0 = full close)

*** order update-stop — Update stop-loss and/or take-profit on an open trade

Update the stop-loss and/or take-profit price on an open trade.  Use 0
to leave a value unchanged or a negative value to cancel it.

#+BEGIN_EXAMPLE
trader order update-stop --trade-id 12345 --stop 1.08200
trader order update-stop --trade-id 12345 --stop 1.08200 --take 1.09500
trader order update-stop --trade-id 12345 --take -1   # cancel take-profit
#+END_EXAMPLE

Flags:
- =--trade-id=  Trade ID to update (required)
- =--stop=      New stop-loss price (0=unchanged, <0=cancel)
- =--take=      New take-profit price (0=unchanged, <0=cancel)

*** order prices — Fetch live bid/ask prices for the major pairs

Query OANDA for current bid/ask prices and print a summary table showing
bid, ask, mid, spread in pips, and USD pip value per lot.  Defaults to all
seven major pairs.

Flags:
- =--instruments=  Comma-separated pairs to query (default: all majors)
- =--units=        Lot size for pip value column (default 100000)

*** order transactions — List OANDA account transactions

List recent OANDA account transactions.  Defaults to the 25 most recent.

Flags:
- =--since=  Return transactions with ID > this value
- =--limit=  Max transactions to display (default 25)

*** order transactions-stream — Subscribe to OANDA transaction stream (push)

Subscribe to the OANDA transaction push stream and print events as they
arrive.  Ctrl-C to exit.

Flags:
- =--heartbeats=  Also print heartbeat messages

** replay — Replay tick datasets through the sim engine

Replay CSV tick data through the simulation engine for testing fills,
journal output, and event scripting.

*** replay pricing — Replay pricing ticks from CSV

Replay pricing ticks from a CSV file (columns:
=time,instrument,bid,ask=) through the simulation engine.

Flags:
- =--ticks=             CSV path
- =--from= / =--to=     Optional RFC3339 time bounds
- =--close-end=         Close all open trades at end of replay
- =--starting-balance=  Starting account balance (default 100000)

*** replay events — Replay pricing + scripted events from CSV

Replay pricing ticks interleaved with scripted trade events from a CSV
file (columns: =time,instrument,bid,ask,event,p1,p2,p3,p4=).

Flags:
- =--ticks=             CSV path
- =--from= / =--to=     Optional RFC3339 time bounds
- =--close-end=         Close all open trades at end of replay
- =--starting-balance=  Starting account balance (default 100000)

** review — Parse a ChatGPT forex review CSV and print trade candidates

Parse a CSV exported from a ChatGPT forex review session.  Prints
tradeable and watchlist pairs in a table; suppresses "No Trade" rows by
default.

Flags:
- =--file=  Path to the ChatGPT forex review CSV (required)
- =--all=   Include "No Trade" rows

** serve — Run trader as a long-running daemon (REST API + live journal)

Start trader in daemon mode.  Boots in order:

1. Structured logging
2. DataManager with warm candle cache
3. OANDA broker connection
4. Transaction stream → journal writer (reconnects on disconnect with exponential backoff)
5. REST API server (=:9999= by default)
6. MCP over HTTP at =POST /mcp= (read-only by default)
7. Embedded SvelteKit UI assets
8. Graceful shutdown on SIGTERM / SIGINT

Configuration can be loaded from a YAML file (=--config=) with CLI flags
taking precedence over file values.

#+BEGIN_EXAMPLE
trader serve
trader serve --token $OANDA_TOKEN --account-id 001-123-456789-001
trader serve --config /etc/trader/trader.yaml
#+END_EXAMPLE

Example config file:
#+BEGIN_EXAMPLE
env: practice
token: ""          # or set OANDA_TOKEN
account_id: ""     # auto-discovered if omitted
rest:
  addr: ":9999"
journal:
  kind: json
  trades_path: /var/lib/trader/live-trades.jsonl
  equity_path:  /var/lib/trader/live-equity.jsonl
log:
  level: info
#+END_EXAMPLE

Flags:
- =--addr=              REST API listen address (default =:9999=)
- =--config=            Path to YAML config file
- =--env=               OANDA environment: practice|live (default practice)
- =--token=             OANDA API token
- =--account-id=        OANDA account ID
- =--journal-trades=    Journal trade-record path (default =./live-trades.jsonl=)
- =--journal-equity=    Journal equity-record path (default =./live-equity.jsonl=)
- =--reports-dir=       Backtest reports directory
- =--mcp-enable-write=  Enable MCP write tools over HTTP (no auth — restrict network access)

** version — Print version information

Print the version, git commit, and build date of the trader binary.
